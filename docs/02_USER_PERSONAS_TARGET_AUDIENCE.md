# User Personas and Target Audience

## Primary audience

Small agencies, solo operators, niche site owners, SEO consultants, and content-heavy businesses that need repeatable AI-assisted blog production.

## Persona 1: Solo SEO operator

### Profile

Runs several niche blogs or client blogs. Wants speed, repeatability, and lightweight publishing.

### Needs

- Create repeatable article pipelines.
- Tune prompts per site/client.
- Save outputs for audit/reuse.
- Push drafts to WordPress.

### Pain points

- One-shot AI drafts feel generic.
- Managing prompts manually is messy.
- Publishing and asset organization are repetitive.

## Persona 2: Small agency owner

### Profile

Manages content for multiple client accounts. Needs separation by client/brand.

### Needs

- Accounts for clients.
- Pipelines per account.
- Run history and published locations.
- Simple pause/start controls.

### Pain points

- Client-specific tone/rules get lost.
- Hard to track what was generated and posted.
- Team could accidentally publish to wrong destination.

## Persona 3: Content systems builder

### Profile

Power user inspired by automation tools. Wants composable agents but not necessarily a visual canvas.

### Needs

- Ordered agents.
- Variable injection.
- Prior agent output references.
- Clear run output inspection.

### Pain points

- Most writing apps hide the workflow.
- Automation tools are too broad/heavy for content workflows.

## Persona 4: WordPress publisher

### Profile

Operates one or more WordPress sites and needs drafts created consistently.

### Needs

- Title, excerpt, thumbnail, body.
- Draft/live mode.
- Posting status and URL.
- Local backup of generated assets.

### Pain points

- Copy/paste workflow is tedious.
- Image prompts and article docs scatter across folders.

## Audience assumptions

- Users are comfortable with prompts, but not necessarily with code.
- Users understand WordPress basics.
- Users value direct controls over conversational chat UX.
- Users prefer visible structure and saved history.
